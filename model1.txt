# Model 1: Conv1D with Token Embeddings - Complete Explanation

## Overview
Model 1 is a **1-dimensional Convolutional Neural Network** that processes sequences of text by converting them into token embeddings and then applying convolution operations to extract features. This is our first deep learning approach after the baseline TF-IDF model.

---

## Step-by-Step Breakdown

### Step 1: Define Model Inputs
```python
inputs = layers.Input(shape=(1,), dtype=tf.string)
```
- Creates an input layer that accepts **string data** (raw text)
- Shape `(1,)` means it expects a single string input (the sentence)
- `dtype=tf.string` specifies the input data type is text/string

---

### Step 2: Vectorize the Text
```python
text_vectors = text_vectorizer(inputs)
```
- Takes the raw text string from `inputs`
- Passes it through the `text_vectorizer` we created earlier
- **What text_vectorizer does:**
  - Converts words into numbers (tokenization)
  - Removes punctuation and converts to lowercase (standardization)
  - Pads or truncates sequences to length 55 (output_sequence_length)
- **Output:** A tensor of shape `(batch_size, 55)` containing integers representing word indices

---

### Step 3: Create Token Embeddings
```python
token_embeddings = token_embed(text_vectors)
```
- Takes the numerical token sequences from Step 2
- Passes them through the embedding layer
- **What the embedding layer does:**
  - Each token (word number) gets converted to a dense vector of 128 dimensions
  - Similar words will have similar embedding vectors
  - This is learned during training
- **Output:** A tensor of shape `(batch_size, 55, 128)`
  - 55 tokens per sequence
  - Each token represented as 128-dimensional vector

---

### Step 4: Apply 1D Convolution
```python
x = layers.Conv1D(64, kernel_size=5, padding="same", activation="relu")(token_embeddings)
```
- Applies 1D convolution filters to the embedding
- **Parameters explained:**
  - `64`: Number of convolution filters (learns 64 different patterns)
  - `kernel_size=5`: Each filter looks at 5 consecutive tokens at a time
  - `padding="same"`: Adds padding so output has same sequence length as input
  - `activation="relu"`: ReLU activation function (introduces non-linearity)
- **What it does:** Detects local patterns in the sequence (e.g., common word combinations)
- **Output:** A tensor of shape `(batch_size, 55, 64)`

---

### Step 5: Global Average Pooling
```python
x = layers.GlobalAveragePooling1D()(x)
```
- Takes the sequence of 64-dimensional vectors from Step 4
- **What it does:**
  - Computes the average across all 55 positions in the sequence
  - Reduces the sequence dimension to a single summary vector
  - This is why it's called "global" pooling (considers all positions)
- **Why average pooling?** Summarizes what the model learned from the entire sequence
- **Output:** A tensor of shape `(batch_size, 64)` - just one vector per sample

---

### Step 6: Output Layer
```python
outputs = layers.Dense(num_classes, activation="softmax")(x)
```
- Takes the summary vector from Step 5
- Applies a fully connected (Dense) layer
- **Parameters:**
  - `num_classes=5`: One output neuron per class (OBJECTIVE, METHODS, RESULTS, BACKGROUND, CONCLUSIONS)
  - `activation="softmax"`: Converts outputs to probabilities (sum to 1.0)
- **Output:** A tensor of shape `(batch_size, 5)` with probability scores for each class

---

### Step 7: Create the Model
```python
model_1 = tf.keras.Model(inputs, outputs)
```
- Combines all layers into a complete model
- `inputs`: The raw text strings we defined in Step 1
- `outputs`: The class probabilities from Step 6
- **Complete pipeline:** String → Vectorize → Embed → Conv1D → Pool → Output

---

### Step 8: Compile the Model
```python
model_1.compile(loss="categorical_crossentropy",
                optimizer=tf.keras.optimizers.Adam(),
                metrics=["accuracy"])
```
- **loss="categorical_crossentropy":** For multi-class classification (5 possible labels)
- **optimizer=Adam():** Algorithm to update weights during training (adaptive learning rate)
- **metrics=["accuracy"]:** Track accuracy during training

Class Predictions (OBJECTIVE, METHODS, RESULTS, BACKGROUND, CONCLUSIONS)

## Why This Architecture Works

1. **Text Vectorization:** Converts readable text into numbers the model can process
2. **Token Embeddings:** Captures semantic meaning (related words have similar embeddings)
3. **Conv1D:** Efficiently learns local patterns (5-word windows) relevant to classification
4. **Pooling:** Summarizes the entire sequence into a single meaningful representation
5. **Output Layer:** Makes final classification decision based on learned features

## Common Issues and Solutions

### Issue 1: Sequence Length Mismatch
**Problem:** Variable-length sentences cause tensor shape errors
**Solution:** Text vectorizer automatically pads/truncates to 55 tokens

### Issue 2: Out-of-Vocabulary (OOV) Words
**Problem:** Words not in training vocabulary at test time
**Solution:** Text vectorizer maps unknown words to OOV token (index 1)

### Issue 3: Class Imbalance
**Problem:** Some classes (e.g., OBJECTIVE) appear less frequently
**Solution:** Use weighted loss or collect more balanced data
